# Эксперимент Object Storage: публичный доступ и восстановление

## Цель

Проверить риск несанкционированного раскрытия данных при публичном доступе к бакету и эффективность мер защиты: закрытие публичного доступа и включение версионирования.

## Ресурсы

- Публичный бакет: `cs-demo-public-b1gu3n-06081116`
- Защищенный бакет: `cs-demo-secure-b1gu3n-06081116`
- Тестовый объект: `demo-report.txt`
- Объект для проверки восстановления: `dr-test.txt`

## Состояние «до»

В публичный бакет был загружен тестовый файл `demo-report.txt`. Анонимный HTTP-запрос к объекту возвращает содержимое файла, что демонстрирует риск раскрытия данных при ошибочной настройке публичного доступа.

Evidence:

- `evidence/before/public-bucket-before.json`
- `evidence/before/public-bucket-curl-before.txt`

## Состояние «после»

Для защищенного бакета публичный доступ не включался. Анонимный HTTP-запрос к объекту завершается отказом доступа. Дополнительно включено версионирование и выполнена проверка получения предыдущей версии объекта.

Evidence:

- `evidence/after/secure-bucket-after.json`
- `evidence/after/secure-bucket-versioning-after.json`
- `evidence/after/secure-bucket-curl-after.txt`
- `evidence/after/secure-bucket-object-versions-after.json`
- `evidence/after/restored-object-content-after.txt`

## Вывод

Эксперимент показал, что публичный доступ к бакету приводит к возможности получения объекта без авторизации. Закрытие публичного доступа и включение версионирования снижают риск несанкционированного раскрытия и потери данных.
